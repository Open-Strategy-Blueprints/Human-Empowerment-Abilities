# 价值锚点识别方法论检查清单

## 📋 检查清单使用说明

本清单用于确保你在使用《价值锚点识别手册》时遵循完整的方法论流程。请在每个步骤完成后勾选对应项。

---

## 🔍 第一阶段：准备与理解（预备工作）

### 1.1 基本概念理解
- [ ] **理解价值锚点的定义**：
  - [ ] 认识到价值锚点是个人或组织中稳定、持久、可扩展的价值核心
  - [ ] 理解四大维度：时间、空间、情感、逻辑
  - [ ] 明确价值锚点与价值观、目标、习惯的区别

### 1.2 工具与材料准备
- [ ] **环境准备**：
  - [ ] 安静、无干扰的思考空间
  - [ ] 纸质笔记本或电子文档
  - [ ] 计时工具（用于限定思考时间）
  
- [ ] **表格准备**：
  - [ ] 打印或复制以下表格到工作空间：
    - [ ] 稳定性评估表格（stability-assessment.md）
    - [ ] 连接性映射表格（connectivity-mapping.md）
    - [ ] 可扩展性验证表格（expansion-verification.md）

### 1.3 心态准备
- [ ] **设定正确心态**：
  - [ ] 保持开放、非评判的态度
  - [ ] 接受不确定性，允许模糊性存在
  - [ ] 预留充足的思考时间（建议至少2小时不被打扰）

---

## 🎯 第二阶段：识别与发现（步骤1-3）

### 2.1 价值元素挖掘
- [ ] **触发问题清单**（至少回答5个）：
  - [ ] "我/我们最引以为豪的三项成就是什么？"
  - [ ] "在困难时期，是什么支撑着我/我们前进？"
  - [ ] "如果只能传承一样东西给下一代，那会是什么？"
  - [ ] "哪些习惯/行为模式反复出现在成功经历中？"
  - [ ] "当需要做重大决定时，我/我们的核心考量因素是什么？"

- [ ] **多维度交叉验证**：
  - [ ] 从以下至少三个角度收集线索：
    - [ ] 历史关键时刻分析
    - [ ] 日常重复行为模式
    - [ ] 他人反馈与评价
    - [ ] 情感强烈反应的事件

### 2.2 初步锚点清单创建
- [ ] **创建候选列表**：
  - [ ] 列出所有潜在价值锚点（建议5-10个）
  - [ ] 使用简洁的短语描述每个锚点（不超过7个字）
  - [ ] 标记每个锚点的情感强度（高/中/低）

- [ ] **初步筛选**：
  - [ ] 移除明显不符合定义的选项
  - [ ] 合并高度相似或重叠的选项
  - [ ] 保留至少3个最有可能的候选锚点

### 2.3 稳定性评估（使用稳定性评估表格）
- [ ] **完成评估表格**：
  - [ ] 为每个候选锚点填写完整表格
  - [ ] 按四个维度分别评分（0-10分）
  - [ ] 记录评分理由和具体证据

- [ ] **计算与比较**：
  - [ ] 使用锚点计算器（anchor-calculator.py）或手动计算
  - [ ] 比较各锚点的总分和维度得分
  - [ ] 识别得分模式（哪个维度普遍强/弱？）

---

## 🔗 第三阶段：连接与验证（步骤4-5）

### 3.1 连接性映射（使用连接性映射表格）
- [ ] **建立连接网络**：
  - [ ] 为中心锚点建立至少5个连接
  - [ ] 区分直接连接与间接连接
  - [ ] 标注连接强度和类型（支持/制约/转化）

- [ ] **网络分析**：
  - [ ] 识别关键连接节点
  - [ ] 检查网络冗余度（是否过于依赖单一连接？）
  - [ ] 寻找潜在的新连接机会

### 3.2 可扩展性验证（使用可扩展性验证表格）
- [ ] **场景测试**：
  - [ ] 设计至少3个扩展应用场景
  - [ ] 模拟不同规模的应用（个人/团队/组织）
  - [ ] 考虑时间跨度（1年/3年/10年）

- [ ] **风险评估**：
  - [ ] 识别潜在扩展障碍
  - [ ] 评估资源需求与实际可用性
  - [ ] 制定缓解策略

### 3.3 交叉验证
- [ ] **多角度验证**：
  - [ ] 寻求外部视角验证（与信任的人讨论）
  - [ ] 检查历史一致性（过去是否一直有效？）
  - [ ] 测试极限情况（在极端压力下是否依然有效？）

- [ ] **整合反馈**：
  - [ ] 记录所有验证结果
  - [ ] 调整或完善锚点定义
  - [ ] 明确适用范围和边界条件

---

## 📈 第四阶段：应用与优化（步骤6-7）

### 4.1 应用策略制定
- [ ] **应用场景明确**：
  - [ ] 确定3个立即可以应用的具体场景
  - [ ] 制定应用时间表和里程碑
  - [ ] 明确成功指标和评估方法

- [ ] **资源分配**：
  - [ ] 识别支持应用的关键资源
  - [ ] 制定资源获取或培养计划
  - [ ] 建立监控和调整机制

### 4.2 实践练习完成（至少完成1项练习）
- [ ] **选择练习**：
  - [ ] [ ] 初级：老照片分析练习（practice-1-photos.md）
  - [ ] [ ] 中级：汉字分析练习（practice-2-characters.md）
  - [ ] [ ] 高级：技艺传承分析练习（practice-3-skills.md）

- [ ] **练习执行**：
  - [ ] 按照练习指南完整完成
  - [ ] 记录过程中的洞察和发现
  - [ ] 对比练习前后对价值锚点的理解

### 4.3 持续优化系统
- [ ] **建立反馈循环**：
  - [ ] 设置定期回顾时间（建议每季度一次）
  - [ ] 创建简单的评估工具或模板
  - [ ] 建立调整决策流程

- [ ] **知识管理**：
  - [ ] 整理所有工作文档和表格
  - [ ] 创建关键洞察摘要
  - [ ] 规划知识传递或分享计划

---

## 📊 第五阶段：审查与文档

### 5.1 完整审查
- [ ] **方法论完整性检查**：
  - [ ] 确认所有必需步骤已完成
  - [ ] 检查各阶段之间的逻辑连贯性
  - [ ] 验证关键决策点的记录

- [ ] **成果质量评估**：
  - [ ] 评估最终锚点的清晰度和实用性
  - [ ] 确认支持证据的充分性
  - [ ] 检查应用计划的可行性

### 5.2 文档整理
- [ ] **核心文档创建**：
  - [ ] 编写价值锚点定义说明书
  - [ ] 创建应用案例集
  - [ ] 整理常见问题解答

- [ ] **版本管理**：
  - [ ] 标注当前版本号和日期
  - [ ] 记录重大修改和原因
  - [ ] 规划下一次审查时间

### 5.3 分享与传播（可选）
- [ ] **内部分享**：
  - [ ] 准备简短的介绍演示
  - [ ] 组织小范围讨论会
  - [ ] 收集反馈用于改进

- [ ] **外部分享考虑**：
  - [ ] 评估分享的可能性和风险
  - [ ] 准备适当抽象程度的版本
  - [ ] 选择合适的分享渠道和方式

## 📝 注意事项

### 常见陷阱提醒
- ❌ **避免过早判断**：在收集阶段不要过滤想法
- ❌ **避免完美主义**：接受"足够好"而非"完美"
- ❌ **避免孤立思考**：适当寻求外部视角
- ❌ **避免静态思维**：价值锚点可能随环境变化

### 灵活调整原则
- ✅ **根据实际情况调整步骤顺序**
- ✅ **合并或拆分步骤以适应具体需求**
- ✅ **适当简化或深化某些环节**
- ✅ **保留调整记录以供后续参考**

### 质量保证要点
- ✅ **证据导向**：每个判断都有具体证据支持
- ✅ **逻辑连贯**：各阶段结论相互支持
- ✅ **实用为本**：最终产出要能实际应用
- ✅ **持续改进**：建立定期审查机制

## 📋 文件说明

### **文件定位**：
- **位置**：`tools/methodology-checklist.md`
- **作用**：方法论执行的系统性检查工具
- **特点**：交互式、可勾选、结构化

### **使用建议**：
1. **打印使用**：建议打印纸质版，方便勾选和记录
2. **电子使用**：可在Markdown编辑器中直接勾选（支持预览模式）
3. **团队使用**：可多人协作，每人负责不同部分的检查
4. **培训使用**：作为新手的引导工具，确保不遗漏关键步骤

### **与其他文件的关系**：
- **引用表格**：直接关联 `tables/` 目录下的所有表格
- **指导练习**：明确指向 `exercises/` 目录下的练习文件
- **补充手册**：作为 `handbook.md` 中方法论部分的实践指南
